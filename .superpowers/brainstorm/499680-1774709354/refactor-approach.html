Refactoring Strategy

How should we structure the project for the future?

A

Full Professional Package

Uses pyproject.toml and src/. Installable via pip. Best for distribution via PyPI or internal tools. Cleanest code separation.

  • Industry standard
  • Easy dependencies
  • Harder to "just copy" folder
B

Hybrid (Modular + Bundler)

We develop in modules (clean), but have a script to bundle it back into a single xyz-test-runner.py for easy sharing.

  • Best of both worlds
  • Stays portable
  • Extra build step needed
C

Simple Modular Split

Just split the 1.6k lines into 3-4 files in the root folder. No complex structure, just xyz_main.py, xyz_engine.py, etc.

  • Zero overhead
  • Very easy to read
  • Not a "standard" package
